上游 算力基础设施层

AI 的起点是算力。一张 GPU 的晶体管数量、内存带宽、互联速率，直接决定了大模型能训多大、推理多快。整个上游基础设施围绕一个核心命题：如何在单位时间内、以最低成本，把最多的算力喂给模型。

2.1 AI 芯片（GPU / ASIC / NPU）

AI 芯片是整个产业链的"发动机"。当前格局：NVIDIA 一家独大（训练市场 90%+），但云厂商自研 ASIC 和国产替代正在加速。

2.2 HBM（高带宽存储器）

HBM 是 AI 芯片的"血液"——GPU 算力再强，数据喂不进去也是白搭。HBM 通过 3D 堆叠 DRAM die + TSV 硅穿孔技术，实现远超传统 DDR 的带宽。

2.3 先进封装（CoWoS / InFO / 3D IC）

先进封装是把 GPU die + HBM + I/O 芯片"拼"在一起的关键技术。没有先进封装，HBM 堆再多也无法与 GPU 高速通信。

封装技术全称说明代表公司应用场景
CoWoS Chip-on-Wafer-on-Substrate TSMC 核心技术：将 GPU die 和 HBM 通过硅中介层（Silicon Interposer）连接，再焊接到有机基板上 TSMC（垄断） NVIDIA H100/B200 全系列
InFO Integrated Fan-Out 扇出型封装，无需硅中介层，成本更低，适合中端产品 TSMC Apple M 系列、部分推理芯片
EMIB Embedded Multi-die Interconnect Bridge Intel 技术：在基板内嵌入小硅片实现 die 间互连，无需整块硅中介层 Intel Ponte Vecchio GPU
Foveros 3D 堆叠封装 Intel 的 3D 堆叠技术，die 直接面对面键合 Intel Core Ultra 系列
Fan-Out WLP Fan-Out Wafer Level Package 晶圆级扇出封装，适合高密度 I/O 长电科技、通富微电、华天科技 移动芯片、中端 AI 芯片
核心瓶颈：TSMC CoWoS 产能是当前 AI 芯片交付的最大瓶颈。2025 年 CoWoS 月产能约 4 万片，仍无法满足 NVIDIA/AMD 的全部订单。TSMC 正在竹南/龙潭扩产，预计 2026 年月产能翻倍。国内先进封装（长电/通富）在 2.5D 封装领域有突破，但与 CoWoS 的技术代差约 2-3 年。

2.4 光模块（800G / 1.6T / CPO）

光模块是 AI 集群中 GPU 服务器之间互联的"高速公路"。一块 GPU 的算力再强，数据传输慢了就是瓶颈。AI 集群对光模块的需求是传统数据中心的 10-50 倍。

技术代际速率核心器件技术难点代表公司

800G（当前放量） 800 Gbps EML/硅光 + 7nm DSP 信号完整性（PAM4 调制）；功耗（~18W）；热管理 中际旭创（出货第一）、新易盛、Coherent、Lumentum

2.5 网络设备（交换机 / RDMA / InfiniBand）

万卡/十万卡 GPU 集群的核心挑战不是单卡算力，而是通信效率。训练大模型时，每张 GPU 都需要频繁地与其他 GPU 同步梯度，网络带宽和延迟直接决定了集群的"有效算力"。

技术说明带宽延迟公司
InfiniBand NVIDIA 旗下高性能网络协议，AI 训练集群的"标配" NDR 400Gb/s → XDR 800Gb/s ~0.6μs NVIDIA（收购 Mellanox，垄断）
RoCE v2 基于以太网的 RDMA 协议，成本比 IB 低 30-50% 400Gb/s → 800Gb/s ~1.5μs Broadcom、华为、中兴、锐捷网络
UALink AMD/Intel/Google 主导的开放 GPU 互联标准，对标 NVLink 200Gb/s → 400Gb/s ~1μs AMD、Intel、Broadcom（联盟）
交换芯片 网络交换机的核心 ASIC Broadcom Tomahawk 5: 51.2Tbps 纳秒级 Broadcom（垄断）、盛科通信、中兴微
核心矛盾：NVIDIA 同时垄断 GPU + NVLink + InfiniBand，形成了从芯片到网络的垂直整合。竞争对手（AMD/Intel/Google）联合推出 UALink + Ultra Ethernet 试图打破封锁，但生态建设需要时间。

2.6 液冷散热

单台 AI 服务器（8 卡）功耗从 A100 的 ~6.5kW 飙升到 GB200 NVL72 的 ~120kW，传统风冷已完全无法应对。液冷从"可选"变成"刚需"。

中游 模型与平台层

中游是 AI 的"大脑"。大模型的能力上限由三个要素决定：架构设计（怎么组织参数）、训练数据（用什么教模型）、训练方法（怎么教模型）。

MoE（Mixture of Experts）架构 — 当前最重要的架构创新

MoE 是 DeepSeek、Mixtral、Qwen、腾讯混元等模型的核心架构。思路很简单：模型有几百个"专家"子网络，但每次推理只激活其中一小部分（如 8/128），从而用大模型的能力但只付出小模型的计算成本。

3.4 推理优化（让模型跑得更快更便宜）

推理成本是 AI 大规模商业化的关键瓶颈。一次 GPT-4 级别的对话，2023 年成本约 $0.1，2025 年降至 $0.001-0.01，降幅超 90%。这背后是多项推理优化技术的叠加。

技术原理效果难点公司
量化（Quantization） 将 FP16 权重压缩到 INT8/INT4，减少内存和计算量 内存减半，速度提升 1.5-2x 低比特（INT4）精度损失；对不同层的敏感度不同 GPTQ、AWQ、SmoothQuant
KV Cache 优化 缓存已计算的 Key/Value 向量，避免重复计算 PagedAttention（vLLM）减少内存碎片 50%+ 长序列 KV Cache 内存占用大；多请求共享 KV Cache 的策略 vLLM、TensorRT-LLM、SGLang
推测解码（Speculative Decoding） 用小模型快速"猜"多个 token，大模型一次性验证 推理速度提升 2-3x 小模型猜中率决定效果；实现复杂度 Google、DeepSeek
模型蒸馏 用大模型（教师）的输出训练小模型（学生） 7B 模型可获得 70B 模型 80-90% 的能力 能力损失不可控；蒸馏数据质量依赖教师模型 Google、Meta、DeepSeek
推理芯片专用化 为推理场景定制的 ASIC，去掉训练不需要的功能 Groq LPU 推理速度 500+ tokens/s（vs GPU 50-100） 灵活性差（只能跑特定模型）；生态薄弱 Groq、Cerebras、寒武纪

3.5 AI Agent（智能体）

Agent 是 2025-2026 年 AI 应用的核心方向。从"模型回答问题"进化为"模型自主完成任务"。

Agent 技术栈

能力层说明技术难点代表产品
规划（Planning） 将复杂任务拆解为子任务序列（如 ReAct、CoT、Tree-of-Thought） 长链规划错误累积（做 10 步以上容易跑偏）；回溯和纠错能力弱 OpenAI o3/o4、DeepSeek-R1
工具调用（Tool Use） 调用 API、数据库、代码执行器、浏览器等外部工具 函数描述格式标准化；多工具编排的错误处理；安全沙箱 OpenAI Function Calling、Anthropic Tool Use、Coze
记忆（Memory） 短期（对话上下文）+ 长期（向量数据库存储的历史交互） 长期记忆检索的准确率；记忆冲突解决；隐私保护 MemGPT、Dify
多 Agent 协作 多个 Agent 分工协作完成复杂任务 Agent 间通信协议；任务分配策略；死锁和循环问题 AutoGen（微软）、CrewAI、Coze 多 Agent
计算机使用（Computer Use） Agent 直接操作电脑界面（鼠标点击、键盘输入、屏幕截图理解） UI 元素识别准确性；操作延迟；跨平台兼容 Anthropic Computer Use、OpenAI Operator、智谱 AutoGLM
